الأمان والمصادقة

تسجيل الدخول

يعتمد النظام على Supabase Auth باستخدام البريد الإلكتروني وكلمة المرور المشفرة (Bcrypt).

  • 🔑 Auto-Login: يتم استعادة الجلسة تلقائياً إذا كانت صالحة.
  • 🕒 Auto-Refresh: تحديث الصفحة يومياً الساعة 3 صباحاً لجلب الجدول الجديد.
  • 🔒 RLS Policies: كل موظف يرى بياناته فقط وبيانات الفريق العامة.

مستويات الصلاحيات

Admin Portal
يستخدم service_role Key (صلاحيات كاملة للقراءة والكتابة والتعديل).
Agent Portal
يستخدم anon Key + Auth Token (صلاحيات مقيدة بـ RLS).

التحديات والحلول (Success Stories)

✅ All Resolved

تم حل جميع المشاكل التقنية التي واجهت الإصدارات السابقة، ليصبح النظام مستقراً 100%.

✔
مشكلة التوقيت (Timezone Mismatch)
التحدي: TODAY'S STATUS كان يجلب بيانات اليوم السابق بسبب فرق التوقيت.
الحل: توحيد منطق التواريخ عبر دالة getLocalDateStr() للتأكد من تطابق توقيت الخادم مع توقيت القاهرة (UTC+3).
✔
ثغرات الأمان (Security Flaws)
التحدي: استخدام كلمات مرور كنص صريح وغياب RLS.
الحل: الانتقال الكامل لـ Supabase Auth مع تشفير كلمات المرور وتفعيل Row Level Security على جميع الجداول.
✔
بيانات الأداء (KPIs Calculation)
التحدي: الاعتماد على جدول وسيط غير دقيق.
الحل: إعادة كتابة محرك الحساب ليجلب البيانات مباشرة من المصدر (daily_performance, call_logs) بدقة فورية.
05 / 07

تدفق البيانات (Data Flow)

نظام تفاعلي: الأدمن يدير البيانات، والموظف يستهلكها ويساهم فيها.

1. إدارة الجدول (Schedule & Breaks)
Admin Portal يقوم بنشر الشيفتات → النظام يقوم بتوليد البريكات تلقائياً → Agent Portal يعرضها فوراً.
2. سجل المكالمات (Call Logs)
Agent يسجل المكالمة (Online/Offline) → يتم حفظها في call_logs → تظهر في تقارير الأداء (KPIs).
3. الطلبات (Requests & Excuses)
Agent يرفع طلب → يظهر كـ Pending → Admin يعتمده أو يرفضه → تتحديث الحالة للطرفين.
