Login & Authentication
إزاي الـ agent بيسجل دخول وبتتحقق هويته
الـ agent بيكتب اسمه وباسورده
←
جيب email من agents table
←
sbClient.auth.signInWithPassword()
←
window._authToken = session.access_token
←
showDashboard()
الموقفاللي بيحصل
أول تحميل — فيه session محفوظةبيعمل sbClient.auth.getSession() — لو صحيحة يدخل مباشرة
أول تحميل — مفيش sessionبيعرض شاشة Login
Login ناجحبيحفظ ns-session في sessionStorage + window._authToken
Logoutبيمسح الـ session + sbClient.auth.signOut() + بيوقف كل الـ timers
Back buttonبيظهر confirm logout dialog
3 صبحauto reload للصفحة
ملاحظة: الـ agent بيدخل باسمه (مش email). الكود بيجيب الـ email من جدول agents بناءً على الاسم، وبعدين بيستخدمه مع Supabase Auth.
لو الـ agent مفيش عنده email في الجدول — بيتعمله email تلقائي: firstname.lastname@nos.internal
تغيير الباسورد: متاح من مكانين:
  • من الـ Dashboard — أيقونة القفل في الـ side menu → submitChangePassword()
  • من شاشة Login — "Forgot Password" → submitReset()
  • الاتنين بيستخدموا sbClient.auth.updateUser({ password }) + بيحدّثوا password_hash في agents table
Supabase — الجداول والصلاحيات
كل جدول بيتعامل معاه الـ agent portal وإيه اللي بيعمله فيه
الجدول الـ Agent Portal RLS Policy الغرض
agents READ UPDATE كل الـ agents مرئيين — بيعدّل بتاعه بس بيجيب قائمة الـ agents للـ dropdowns، وبيحدّث الباسورد
schedule READ كل الـ schedule مرئي للكل شيفت الأسبوع الحالي والجاي، TODAY'S STATUS
schedule_weeks READ Published weeks فقط بيجيب الأسابيع المنشورة عشان يعرض الـ schedule
breaks READ UPDATE INSERT SELECT للكل — UPDATE/INSERT بتاعه بس Team Availability، Today's breaks، تغيير status الـ break
daily_performance READ بتاعه بس KPIs: Conformance, Missing Time, AHT
excuses READ/WRITE بتاعه بس يشوف excuses بتاعته، يرفع excuse جديدة، يتراجع
annual_leave READ بتاعه بس Annual Left/Used في الـ KPI cards وتفاصيل الإجازة
quality_scores READ بتاعه بس Quality % في الـ KPI cards
requests READ/WRITE بتاعه بس يشوف طلباته، يرفع طلب جديد (Time off, Missing punch, Schedule, Shift swap)
call_logs READ/WRITE بتاعه بس يسجل مكالمات، يشوف سجل مكالماته، البحث عن عملاء
shift_types READ public read أوقات الشيفتات عشان يعرض الـ schedule صح
call_reasons READ public read قائمة أسباب المكالمات في الـ call log form
مهم: الـ Admin Portal بيستخدم service_role key — بيـ bypass الـ RLS كلها. الـ Agent Portal بيستخدم anon key مع Supabase Auth token — الـ RLS بتحمي كل agent يشوف بتاعه بس.

Data Flow — مين بيكتب ومين بيقرا
الـ Admin Portal هو اللي بيكتب البيانات، والـ Agent Portal بيقراها
البياناتمين بيكتبهامين بيقراهاالجدول
الـ Schedule Admin → Schedule page Agent → Schedule tab, Dashboard schedule + schedule_weeks
الـ Breaks Admin → Breaks page (auto-generate عند Publish) Agent → Dashboard (Team Availability + Today's Status) breaks
Performance KPIs Admin → xCally Import (من ملف CSV) Agent → Dashboard (Conformance, AHT, Missing) daily_performance
Quality Score Admin → Quality page Agent → Dashboard (Quality card) quality_scores
Annual Leave Admin → Schedule (عند Publish يحدث تلقائي) Agent → Dashboard (Annual Left) annual_leave
Excuses Agent يرفعها ← Admin يعتمدها Agent → يشوف بتاعته — Admin → يعتمد أو يرفض excuses
Requests Agent يرفعها ← Admin يعتمدها Agent → يشوف بتاعته — Admin → يعتمد أو يرفض requests
Call Logs Agent يسجّلها Agent → My Call Log — Admin → Call Log page call_logs
Knowledge Base Admin → KB page (Firebase) Agent → Knowledge Base tab Firebase Firestore

المشاكل الموجودة والحالة
كل مشكلة وحالتها الحالية
المشكلةالسببالحالة
TODAY'S STATUS بيجيب بيانات أمس الـ breaks table بتجيب breaks النهارده بالـ UTC date، بس في Cairo UTC+3 فبيحصل mismatch في بعض الأوقات ⚠️ قيد الحل — الـ fix موجود في getLocalDateStr() بس لسه بيتختبر
GAS_URL موجود في الكود بقايا من الكود القديم ✅ تم الحل — مفيش gasRun calls شغّالة
KPIs كانت بتجيب من kpis table كانت changeMonthData بتقرا من جدول وسيط ✅ تم الحل — بتحسب من المصدر مباشرة
_applyGASBreaks غير موجودة كانت function بتجيب البيانات من GAS وبقت undefined ✅ تم الحل — استُبدلت بـ Supabase query
Login كان يستخدم password_hash مباشرة الباسورد كان stored plain text ومش secure ✅ تم الحل — Supabase Auth مع bcrypt
مفيش RLS أي agent كان يقدر يقرا بيانات أي حد تاني ✅ تم الحل — RLS مفعّل على كل الجداول
Admin Portal بيستخدم anon key نفس صلاحيات الـ agent ✅ تم الحل — Admin بيستخدم service_role
⚠️ المشكلة الوحيدة المتبقية: TODAY'S STATUS أحياناً بيجيب شيفت أمس. ده بيحصل لأن جدول breaks بيـ store الـ break_date بـ format مختلف أو لأن الـ timezone مش متطابق بين ما بيتحفظ في الـ admin وما بيتقرا في الـ agent. الـ fix الكامل هيتأكد لما الـ agent يسجّل دخول وقت الشيفت الحقيقي.
